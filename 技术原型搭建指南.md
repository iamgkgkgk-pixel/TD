## 🚀 步骤一：安装 Unity 2022.3 LTS + WebGL 模块

### 1.1 通过 Unity Hub 安装

1. 打开 Unity Hub → **Installs** → **Install Editor**
2. 选择 **2022.3.xx LTS**（最新patch）
3. **⚠️ 关键**：在模块选择页面，勾选以下模块：
   - ✅ **WebGL Build Support**
   - ✅ **IL2CPP** （通常自带）
4. 点击 Install，等待安装完成

### 1.2 验证安装

```
Unity版本: 2022.3.xx
WebGL模块: 已安装（可在 File → Build Settings → WebGL 看到）
```

## 🚀 步骤二：创建 Unity 空项目

### 2.1 新建项目

1. Unity Hub → **New Project**
2. 选择模板：**2D (URP)** 或 **2D**
   - 推荐 **2D**（不用URP，包体更小，更适合微信小游戏）
3. 项目名称：`AetheraSurvivors`
4. 项目路径：`D:\AIRearch\TD\Unity`
5. 点击 **Create project**

### 2.2 项目结构确认

创建完成后，确认项目结构：

```
├── Assets/
│   ├── Scenes/
│   │   └── SampleScene.unity
│   └── ...
├── Packages/
├── ProjectSettings/
└── ...
```

## 🚀 步骤三：集成 minigame-unity-sdk

### 3.1 下载 SDK

1. 访问 GitHub：https://github.com/nicknightning/aspect-build-rules
   - **正确地址**：https://github.com/nicknightning/aspect-build-rules
   - **⚠️ 实际请用官方地址**：前往微信官方文档获取最新SDK
   
2. **推荐方式**：通过 Unity Package Manager 导入
   - 打开 Unity → **Window** → **Package Manager**
   - 点左上角 **+** → **Add package from git URL...**
   - 输入 minigame-unity-sdk 的 Git URL

### 3.2 替代安装方式（手动导入 .unitypackage）

1. 从微信官方文档下载 `minigame-unity-sdk-x.x.x.unitypackage`
   - 官方文档：https://developers.weixin.qq.com/minigame/dev/guide/game-engine/unity-setup.html
2. Unity 中 **Assets** → **Import Package** → **Custom Package...**
3. 选择下载的 `.unitypackage` 文件
4. **全部勾选** → **Import**

### 3.3 验证 SDK 安装

导入成功后，Unity 菜单栏应出现：
```
菜单栏 → 微信小游戏 （或 MiniGame）
```

如果看到此菜单，SDK 安装成功 ✅

## 🚀 步骤四：部署测试代码

将以下 4 个脚本文件放入 `Assets/Scripts/` 目录：

```
Assets/
└── Scripts/
    ├── WXBridge.cs              -- 微信API桥接层
    ├── TouchTest.cs             -- Touch输入测试
    ├── PerformanceMonitor.cs    -- FPS/内存监控
    └── TechProtoMain.cs         -- 主入口
```

> ⚠️ 这4个文件的代码已在本仓库的 `Unity/Assets/Scripts/` 目录下准备好。

### 4.1 创建测试场景

1. 打开 `Assets/Scenes/SampleScene.unity`（或新建场景 `TechProtoScene`）
2. 创建空 GameObject，命名为 `GameManager`
3. 挂载以下脚本到 `GameManager` 上：
   - `TechProtoMain`
   - `PerformanceMonitor`
   - `TouchTest`
4. 创建一个 **UI Canvas**：
   - 右键 Hierarchy → **UI** → **Canvas**
   - Canvas 下创建 **Text (TMP)** 或 **Text** ，命名为 `InfoText`
   - 设置字体大小 24，颜色白色，位置居中偏上
5. 将 `InfoText` 拖拽赋值给 `TechProtoMain` 的 `infoText` 字段
6. 将 `InfoText` 拖拽赋值给 `PerformanceMonitor` 的 `fpsText` 字段

### 4.2 创建 Touch 可视化

1. Canvas 下创建 **Image**，命名为 `TouchIndicator`
2. 设置大小 80×80，颜色半透明红色 `(1, 0, 0, 0.5)`
3. 默认隐藏（取消勾选 Active）
4. 将 `TouchIndicator` 拖拽赋值给 `TouchTest` 的 `touchIndicator` 字段

## 🚀 步骤五：配置 Build Settings

### 5.1 切换平台

1. **File** → **Build Settings**
2. 选择 **WebGL** → 点击 **Switch Platform**
3. 等待资源重新导入（首次可能需要几分钟）

### 5.2 Player Settings 配置

**File** → **Build Settings** → **Player Settings**：

| 设置项 | 值 | 说明 |
|--------|---|------|
| **Product Name** | AetheraSurvivors | 游戏名称 |
| **Company Name** | YourStudioName | 公司/工作室名 |
| **Resolution → Default Canvas Width** | 750 | 微信标准宽度 |
| **Resolution → Default Canvas Height** | 1334 | 微信标准高度（竖屏） |
| **Other Settings → Color Space** | Gamma | WebGL推荐Gamma |
| **Other Settings → API Compatibility** | .NET Standard 2.1 | — |
| **Other Settings → Managed Stripping Level** | High | 减小包体 |
| **Publishing Settings → Compression Format** | Brotli | 最优压缩 |
| **Publishing Settings → Data Caching** | ✅ 勾选 | — |

### 5.3 微信小游戏导出配置

1. 菜单栏 → **微信小游戏** → **转换小游戏**（或对应菜单）
2. 填写配置：

| 配置项 | 值 | 说明 |
|--------|---|------|
| **AppID** | 你的小游戏AppID（测试可用测试号） | 在微信公众平台注册获取 |
| **项目名称** | AetheraSurvivors | — |
| **导出路径** | `D:\AIRearch\TD\WXExport` | 导出目录 |
| **CDN地址** | 留空（本地测试不需要） | 正式环境需配置 |
| **游戏方向** | 竖屏 Portrait | — |

### 6.1 导出

1. 点击 **导出** 按钮
2. 等待 Unity 编译 + 转换（首次可能 5-15 分钟）
3. 导出完成后，`D:\AIRearch\TD\WXExport` 目录下应有：
```

├── minigame/        ← 这个目录用微信开发者工具打开
│   ├── game.js
│   ├── game.json
│   ├── project.config.json
│   └── ...
├── webgl/           ← Unity WebGL产物
│   ├── Build/
│   └── ...
└── ...
```

### 6.2 微信开发者工具打开

1. 打开 **微信开发者工具**
2. 选择 **小游戏** 项目类型
3. 项目目录选择：`D:\AIRearch\TD\WXExport\minigame`
4. AppID 填写你的测试 AppID
5. 点击 **确定**

### 6.3 首次运行

1. 等待微信开发者工具编译完成
2. 在模拟器中应看到：
   - 屏幕上方显示 FPS/内存信息
   - 中间显示系统信息和登录状态
   - 触摸/点击屏幕会出现红色圆点指示器
   - 控制台（Console）输出详细日志

## 🚨 常见问题排查

### Q1: SDK菜单没出现
- 确认 .unitypackage 导入成功
- 检查 Console 是否有编译错误
- 重启 Unity

### Q2: 导出报错 "IL2CPP not found"
- Unity Hub → Installs → 对应版本 → Add Modules → 勾选 IL2CPP

### Q3: 微信开发者工具白屏
- 检查 project.config.json 中的 appid 是否正确
- 清除缓存：工具栏 → 清缓存 → 全部清除
- 检查 game.js 是否存在

### Q4: Touch 在真机上不响应
- 确认 Canvas 的 Render Mode 为 Screen Space - Overlay
- 确认 EventSystem 存在于场景中

### Q5: FPS 异常低
- 确认 Quality Settings 为 Low/Medium
- 确认没有多余的后处理效果
- 如果使用 URP，考虑切换为 Built-in Render Pipeline

### Q6: wx API 调用失败
- 在 Unity Editor 中运行会失败（正常），**必须在微信开发者工具中测试**
- 确认代码中的条件编译 `#if UNITY_WEBGL && !UNITY_EDITOR` 正确
- 检查 JS 层桥接文件是否被正确导出
