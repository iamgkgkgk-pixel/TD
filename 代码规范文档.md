## 一、命名规范

### 1.1 C# 命名规则

| 类型 | 规则 | 示例 | 反例 |
|------|------|------|------|
| **命名空间** | PascalCase，`AetheraSurvivors.子模块` | `AetheraSurvivors.Framework` | `aethera_survivors.framework` |
| **类/结构体** | PascalCase，名词 | `EnemyBase`, `TowerManager` | `enemyBase`, `tower_manager` |
| **接口** | I + PascalCase | `IPoolable`, `IDamageable` | `Poolable`, `poolable` |
| **公共方法** | PascalCase，动词开头 | `GetDamage()`, `ApplyBuff()` | `getDamage()`, `damage()` |
| **私有方法** | PascalCase，动词开头 | `CalculateDPS()` | `_calculateDPS()` |
| **公共属性** | PascalCase | `MaxHealth`, `AttackRange` | `maxHealth`, `_maxHealth` |
| **公共字段** | PascalCase（尽量用属性代替） | `CurrentWave` | `currentWave` |
| **私有字段** | `_` + camelCase | `_currentHealth`, `_poolDict` | `currentHealth`, `m_health` |
| **局部变量** | camelCase | `enemyCount`, `damageResult` | `EnemyCount`, `_enemyCount` |
| **参数** | camelCase | `targetEnemy`, `damageAmount` | `TargetEnemy`, `_target` |
| **常量** | PascalCase 或 全大写下划线 | `MaxTowerLevel`, `MAX_POOL_SIZE` | `maxTowerLevel` |
| **枚举类型** | PascalCase | `TowerType`, `BuffType` | `towerType`, `TOWER_TYPE` |
| **枚举值** | PascalCase | `TowerType.Archer` | `TowerType.ARCHER` |
| **事件** | PascalCase，名词/动名词 | `EnemyDeathEvent` | `onEnemyDeath` |
| **委托** | PascalCase + Handler/Callback | `DamageHandler` | `damage_handler` |
| **泛型参数** | T + 描述词（单字母T仅在简单场景） | `TPanel`, `TConfig` | `panel`, `t` |

### 1.2 文件命名

| 类型 | 规则 | 示例 |
|------|------|------|
| C#脚本 | 与类名完全一致.cs | `TowerManager.cs` |
| 配置文件 | 小写+下划线.json | `tower_config.json` |
| 预制体 | PascalCase.prefab | `ArcherTower.prefab` |
| 图集 | PascalCase_Atlas | `UI_Common_Atlas` |
| 精灵图 | 模块_名称_状态 | `tower_archer_idle.png` |
| 音效 | sfx_动作_描述 | `sfx_tower_shoot.wav` |
| 背景音乐 | bgm_场景_描述 | `bgm_battle_normal.mp3` |
| 动画 | anim_对象_动作 | `anim_enemy_walk.anim` |

### 1.3 目录命名

- 使用 **PascalCase**（如 `Scripts/Framework/`、`Resources/Configs/`）
- 避免使用空格、中文、特殊字符

```csharp
// 检查金币是否足够

    return false;

// 寻路验证：放塔后路径是否仍然连通
```

### 2.5 注释语言

- **所有注释使用中文**（代码本身使用英文）
- TODO标记格式：`// TODO: [描述] —— [日期]`
- HACK标记格式：`// HACK: [为什么这么做] —— [日期]`
- BUG标记格式：`// BUG: [bug描述] —— [日期]`

### 2.6 什么时候写注释

| 必须写 | 不必写 |
|--------|--------|
| 类/接口的职责说明 | 显而易见的getter/setter |
| 公共方法的参数和返回值 | 自解释的简单赋值 |
| 复杂算法的思路 | 标准的for/if语句 |
| 微信小游戏的平台特殊处理 | 框架生成的模板代码 |
| 性能优化的原因 | — |
| 业务逻辑的"为什么" | — |

## 三、代码组织规范

### 3.1 类成员排列顺序

```csharp
public class ExampleManager : Singleton<ExampleManager>
{
    // ========== 1. 常量 ==========
    private const int MaxCount = 100;

    // ========== 2. 静态字段 ==========
    private static readonly Dictionary<int, string> _sharedCache = new();

    // ========== 3. 序列化字段（Inspector可见） ==========
    [SerializeField] private Transform _spawnPoint;
    [SerializeField] private GameObject _prefab;

    // ========== 4. 私有字段 ==========
    private int _currentCount;
    private List<Enemy> _activeEnemies = new();
    private ObjectPool<Projectile> _projectilePool;

    // ========== 5. 公共属性 ==========
    public int CurrentCount => _currentCount;
    public bool IsActive { get; private set; }

    // ========== 6. Unity生命周期（按调用顺序排列） ==========
    private void Awake() { }
    private void OnEnable() { }
    private void Start() { }
    private void Update() { }
    private void OnDisable() { }
    private void OnDestroy() { }

    // ========== 7. 公共方法 ==========
    public void Initialize() { }
    public void DoSomething() { }

    // ========== 8. 私有方法 ==========
    private void InternalHelper() { }

    // ========== 9. 事件回调 ==========
    private void OnEnemyDeath(EnemyDeathEvent evt) { }

    // ========== 10. 编辑器方法（条件编译） ==========
    #if UNITY_EDITOR
    private void OnDrawGizmos() { }
    #endif
}
```

### 3.2 using排列顺序

```csharp
// 1. System命名空间
using System;
using System.Collections.Generic;

// 2. Unity命名空间
using UnityEngine;
using UnityEngine.UI;

// 3. 第三方库
using LitJson;

// 4. 项目命名空间
using AetheraSurvivors.Framework;
using AetheraSurvivors.Data;
```

### 3.3 单文件单类

- 每个.cs文件只包含一个public类
- 类名必须与文件名一致
- 内部类/嵌套类除外
- 小型数据结构/枚举可以例外（放在同文件底部）

### 3.4 代码格式

| 规则 | 说明 |
|------|------|
| **缩进** | 4个空格（不用Tab） |
| **花括号** | Allman风格（单独一行） |
| **行长度** | 建议不超过120字符 |
| **空行** | 方法之间空1行，逻辑块之间空1行 |
| **region** | 禁止使用`#region`（用注释分隔代替） |

### 3.5 性能编码规范（⚠️ 微信小游戏GC敏感）

| 规则 | 原因 |
|------|------|
| **禁止在Update中new对象** | 产生GC压力 |
| **禁止在循环中拼接string** | 改用StringBuilder |
| **避免使用LINQ** | 产生隐式分配 |
| **避免foreach遍历List** | 在旧版本可能产生GC（用for代替） |
| **高频对象必须走对象池** | 怪物/投射物/特效/飘字 |
| **缓存组件引用** | 不要每帧GetComponent |
| **使用struct传递短生命周期数据** | 减少堆分配 |
| **使用`string.Empty`替代`""`** | 避免创建新字符串对象 |

## 四、架构编码规范

### 4.1 模块间通信

```
✅ 正确：通过 EventBus 发布/订阅事件
❌ 错误：直接引用其他Manager的私有方法
❌ 错误：通过 FindObjectOfType 查找
❌ 错误：通过 static 全局变量传递
```

### 4.2 Manager职责

- 每个Manager有且只有一个明确职责
- Manager之间不能互相持有引用（通过EventBus通信）
- Manager的初始化和销毁由GameManager统一管理

### 4.3 配置与代码分离

```
✅ 正确：怪物属性从JSON配置表读取
❌ 错误：怪物属性硬编码在代码中
✅ 正确：波次配置在JSON中定义
❌ 错误：波次逻辑写死在代码里
```

### 4.4 平台适配

```csharp
// ✅ 正确：条件编译切换平台
#if UNITY_WEBGL && !UNITY_EDITOR
    WXStorageProvider.Save(key, json);
#else
    PlayerPrefs.SetString(key, json);
#endif

// ❌ 错误：运行时判断平台
if (Application.platform == RuntimePlatform.WebGLPlayer)
    // 这也可以，但条件编译更好（编译期剥离无关代码）
```

## 五、Git提交规范

### 5.1 提交信息格式

```
<type>(<scope>): <subject>

<body>（可选）

<footer>（可选）
```

### 5.2 Type类型

| Type | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat(tower): 实现箭塔攻击逻辑` |
| `fix` | Bug修复 | `fix(enemy): 修复寻路终点偏移问题` |
| `refactor` | 重构 | `refactor(pool): 优化对象池扩容策略` |
| `perf` | 性能优化 | `perf(battle): 空间分区加速目标查找` |
| `style` | 代码格式 | `style: 统一缩进为4空格` |
| `docs` | 文档 | `docs: 更新GDD技术约束` |
| `test` | 测试 | `test(eventbus): 添加订阅取消测试` |
| `chore` | 构建/工具 | `chore: 更新.gitignore` |
| `asset` | 资源 | `asset(tower): 添加箭塔图标` |

### 5.3 Scope范围

| Scope | 对应目录 |
|-------|---------|
| `framework` | Scripts/Framework/ |
| `platform` | Scripts/Platform/ |
| `battle` | Scripts/Battle/ |
| `tower` | Scripts/Battle/Tower/ |
| `enemy` | Scripts/Battle/Enemy/ |
| `rune` | Scripts/Battle/Rune/ |
| `meta` | Scripts/MetaGame/ |
| `ui` | Scripts/UI/ |
| `data` | Scripts/Data/ |
| `config` | Resources/Configs/ |
| `editor` | Editor/ |

### 5.4 提交频率

- 每完成一个独立功能提交一次
- 每修复一个Bug提交一次
- 不要积攒大量修改后一次性提交
- 每次提交保证项目可编译

### 5.5 分支策略

```
main          ← 稳定版本，只接受合并
├── develop   ← 开发主分支
│   ├── feature/tower-system   ← 功能分支
│   ├── feature/rune-system
│   └── fix/pathfinding-bug    ← 修复分支
└── release/v1.0               ← 发布分支
```

| 分支 | 用途 | 命名规则 |
|------|------|---------|
| `main` | 生产环境稳定版 | — |
| `develop` | 开发主线 | — |
| `feature/*` | 新功能开发 | `feature/功能名` |
| `fix/*` | Bug修复 | `fix/问题描述` |
| `release/*` | 版本发布 | `release/v版本号` |

## 六、资源规范

### 6.1 图片资源

| 规则 | 说明 |
|------|------|
| **格式** | PNG（含透明）/ JPG（不含透明） |
| **纹理压缩** | ASTC 6x6（导入设置） |
| **最大尺寸** | UI图标≤128px，角色≤256px，地图Tile≤64px |
| **图集** | 同模块UI素材打入同一SpriteAtlas |
| **命名** | `模块_名称_状态.png` |

### 6.2 音频资源

| 规则 | 说明 |
|------|------|
| **BGM格式** | OGG Vorbis，64-128kbps |
| **SFX格式** | WAV/OGG，44.1kHz |
| **加载方式** | BGM流式播放，SFX预加载 |
| **命名** | `bgm_场景.ogg` / `sfx_动作.wav` |

### 6.3 预制体

| 规则 | 说明 |
|------|------|
| **命名** | PascalCase，与主脚本同名 |
| **嵌套** | 预制体嵌套不超过3层 |
| **引用** | 预制体不引用场景中的对象 |

## 七、速查表

```
📁 新建文件 → 类名=文件名，PascalCase
📝 新建类   → 加文件头注释+类注释
🔧 新建方法 → 公共方法加XML注释
📦 新建变量 → 私有用_camelCase，公共用PascalCase
🔌 模块通信 → 只用EventBus，不直接引用
♻️ 高频对象 → 必须走对象池
🚫 Update中 → 禁止new/string拼接/LINQ/GetComponent
📊 Git提交  → type(scope): 中文描述
```
